逻辑层 服务架构详细分析
逻辑层模块是ASCF框架的js运行时核心,提供小程序/快应用的完整运行环境,包括生命周期管理,API桥接,事件系统和模块管理等功能

        PageManager.js 页面管理器
        Component.js 组件基类
        BehaviorManager.js 行为管理器
        bridge
            jsBridge,js js桥接
        module
            wx/ 微信小程序兼容模块
                api api模块

    html5 HTML5相关

核心功能模块
1. 框架初始化
index.js
主要职责
 初始化webApp框架环境
 注册全局对象 App，Page，Component，Behavior
 订阅jsBridge事件
 建立与原生层的通信通道

2. 应用启动器
   service/appLauncher.js 应用启动器
主要职责
    加载并执行小程序的app.js
    解析app.json配置
    初始化应用启动流程

3. 应用管理器
   service/appManager.js 应用管理器
主要职责
    创建和管理App实例
    绑定应用生命周期钩子
    管理应用全局状态
    处理应用级别的时间
生命周期管理
'onLaunch' 应用初始化完成时触发
'onShow' 应用启动或从后台进入前台除非
'onHide'应用从前台进入后台除非
'onError' 应用发生脚本错误时触发

4. 页面管理器
   service/PageManager.js页面管理器
主要职责
   管理页面栈（支持页面导航）
   处理页面路由事件
   管理页面生命周期
   支持异步组件加载
页面栈管理
    维护页面历史记录
    支持navigateTo，redirectTo,navagateBack等路由方法
    处理页面参数传递
页面周期管理
   'onLoad' 页面加载时候触发
   'onShow' 页面显示时触发
   'onReady'页面初次渲染完时触发
   'onHide' 页面隐藏时候触发
   'onUnload'页面卸载时触发

5. 组件系统
   service/Component.js 组件基类
主要职责
   定义组件基类
   管理组件生命周期
   处理组件数据绑定
   管理组件属性和事件
组件生命周期
   'created' 组件实例化
   'attached' 组件挂载
   'ready' 组件渲染完成
   'moved' 组件移动
    'detached' 组件卸载
6. 行为管理器
   service/BehaviorManager.js 行为管理器
主要职责
   实现Behavior特性
   支持Behavior混入
   管理Behavior生命周期
   处理Behavior数据和方法的合并
7. js桥接
   service/bridge
   jsBridge,js js桥接
主要职责
   实现js与原生层的通信
   实现异步调用机制
   管理回调函数
   处理跨语言数据转换
通信机制
   使用消息队列进行通信
   支持同步和异步调用
   自动序列化/反序列化数据
8. 模块管理
ascf的模块管理主要负责提供各类API的定义和部分实现，兼容微信小程序API,降低小程序代码迁移成本。
8.1 API模块结构
目录结构
module /wx/ 微信小程序兼容模块
   api api模块
   base 基础模块
   device 设备模块
   media 媒体模块
   network 网络模块
   service 服务模块
   ui ui模块
   AI AI能力模块
   interface.js API统一导出入口
   nativeComponentEvent.js 原始组件事件
   utils 工具模块
9. API注册
    service/api/register.js
    主要职责
    注册所有可用的api
    建立api名称与实现的映射
    提供api查询接口
10. 与其他模块的交互
    和核心模块交互
    通过JSBride与原生层通信
    调用核心模块提供的API
    接收原生层的事件通知
    与视图层的交互
    提供数据给视图层渲染
    接收视图层用户交互事件
    协调页面和组件的声明周期
11. 添加新的API
  1.首先在wx对应的子目录下创建新的API文件
  2.实现具体的功能逻辑
  3,在interface中导入新的API
  4. 将新API添加到hasInterfaceList对象中
