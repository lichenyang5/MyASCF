架构分析
ASCF（Atomic Service Cross Framework）是元服务为小程序生态定制的一套解决方案，采用应用间HSP模式提供系统级的运行时能力
runtime
├── 核心模块
│   ├── 应用生命周期管理
│   │   ├── UIAbility
│   │   └── AbilityStage
│   │
│   ├── 运行时核心能力
│   │   ├── 页面路由
│   │   ├── 事件管理
│   │   ├── 桥接通信
│   │   └── Web 容器加载
│   │
│   ├── 原生能力 API 实现
│   │   ├── 设备
│   │   ├── 网络
│   │   ├── 媒体
│   │   ├── 存储
│   │   └── 其他系统能力
│   │
│   └── 同层渲染增强
│       ├── video
│       ├── map
│       ├── camera
│       └── canvas
│
├── 逻辑层
│   ├── JS 逻辑运行环境
│   ├── app / page / component 生命周期
│   ├── JS 调原生能力的桥接
│   ├── 事件订阅与发布
│   └── 小程序兼容 API 定义
│
├── 视图层
│   ├── 基于 Vue 扩展的视图框架
│   ├── 基础 UI 组件
│   │   ├── view
│   │   ├── text
│   │   ├── image
│   │   ├── video
│   │   └── 其他组件
│   ├── 样式绑定与管理
│   ├── 用户交互事件处理
│   └── 鸿蒙 WebUI 环境适配
│
└── ohpm
├── 包发布能力
├── HAR 包装
└── 第三方应用集成依赖

构建流程
首先编译 逻辑层和视图层 分别为两个js
然后编译核心模块为一个hsp包
