# ASCF 核心模块结构分析

## 一、整体说明

ASCF 核心模块本身是一个鸿蒙项目，整体以 **HSP** 形式提供运行时能力。

它主要承担以下职责：

* 提供应用生命周期管理能力
* 提供 Web 容器加载页面能力
* 提供 ASCF 运行时核心能力
* 提供逻辑层与视图层之间的通信能力
* 提供各类 API 的具体实现
* 提供同层渲染组件增强能力
* 将核心能力导出给其他应用使用

---

## 二、主模块 HSP

### 目录结构

```text
entry 主模块
└── src/main/ets/
    ├── context        # 上下文管理
    │   ├── AbilityState.ets
    │   └── UIAbility.ets
    │
    ├── pages          # 页面
    ├── component      # 组件
    └── manager        # 管理器
```

### 主要功能

主模块是 HSP 格式的核心模块，主要负责 ASCF 在鸿蒙侧的基础承载能力。

它的主要功能包括：

1. 提供应用生命周期管理能力。
2. 提供 Web 容器加载页面能力。
3. 导出核心功能给其他应用使用。
4. 作为 ASCF Runtime 的鸿蒙侧入口模块。

---

## 三、运行时核心

### 目录结构

```text
runtime 运行时核心
└── src/main/ets/
    ├── api           # API 模块
    ├── bridge        # 桥接层
    ├── event         # 事件处理
    ├── launcher      # 启动器
    ├── manifest      # 清单管理
    ├── package       # 包管理
    ├── preload       # 预加载
    └── route         # 路由
        ├── page      # 页面路由
        ├── tabBar    # 标签栏
        └── webView   # WebView 管理
```

### 主要功能

运行时核心负责提供 ASCF Runtime 的核心调度能力。

它的主要功能包括：

1. 页面路由管理。
2. `app.js` 加载。
3. 分包管理。
4. 配置文件加载。
5. WebView 管理。
6. 通信桥接。
7. JS 与 ArkTS 之间的通信。
8. 事件处理。
9. 启动流程管理。

可以理解为：

```text
应用启动
  ↓
启动运行时
  ↓
加载配置文件
  ↓
加载 app.js
  ↓
初始化页面路由
  ↓
创建或管理 WebView
  ↓
建立 JS 与 ArkTS 通信桥接
  ↓
页面开始运行
```

## 四、组件库

### 目录结构

```text
components 组件库
└── src/main/ets/
    ├── sameLayerComponents   # 同层组件
    ├── 视频组件
    ├── 地图组件
    ├── 相机组件
    ├── apiComponents         # API 组件
    ├── stackComponent        # 栈组件
    └── webComponents         # Web 组件
```

### 主要功能

组件库主要用于提供 ASCF 的组件增强能力，尤其是同层渲染相关能力。

它的主要功能包括：

1. 使用原生组件提供同层渲染能力。
2. 支持视频组件能力。
3. 支持地图组件能力。
4. 支持相机组件能力。
5. 支持 Canvas 等复杂组件能力。
6. 提供 Web 组件相关能力。
7. 支持部分 API 组件化实现。

同层渲染组件主要解决 Web 容器中部分复杂组件能力不足的问题，例如：

```text
video
map
camera
canvas
```

这些组件通常需要借助鸿蒙原生组件能力来实现更好的性能和交互体验。

---

## 五、API 能力模块

### 目录结构

```text
apis
└── src/main/ets/
    ├── base       # 基础 API
    ├── device     # 设备 API
    ├── media      # 媒体 API
    ├── network    # 网络 API
    ├── storage    # 存储 API
    ├── ui         # UI API
    └── openapi    # 开放能力 API
```

### 主要功能

API 能力模块负责提供各类 API 接口的具体实现。

它的主要功能包括：

1. 提供基础 API。
2. 提供设备相关 API。
3. 提供媒体相关 API。
4. 提供网络相关 API。
5. 提供存储相关 API。
6. 提供 UI 相关 API。
7. 提供开放能力 API。
8. 通过桥接鸿蒙原生能力的方式实现具体功能。

整体调用链路可以理解为：

```text
JS 调用 API
  ↓
桥接层接收调用
  ↓
API 模块进行分发
  ↓
调用鸿蒙原生能力
  ↓
封装执行结果
  ↓
返回给 JS
```

---

## 六、原生桥接模块

### 模块定位

原生桥接模块主要负责提供 JS 运行环境和双线程模型桥接能力。

### 主要功能

它的主要功能包括：

1. 提供 V8 运行环境。
2. 支撑逻辑层运行 JavaScript。
3. 提供双线程模型桥接能力。
4. 支持逻辑层和视图层之间的通信。
5. 支持 JS 与 ArkTS 之间的通信。
6. 使用 C++ 实现部分高性能 API。

可以理解为：

```text
逻辑层 JS
  ↓
桥接协议
  ↓
原生桥接模块
  ↓
ArkTS / C++ 能力
  ↓
鸿蒙系统能力
```

这一层是 ASCF Runtime 中非常关键的一层，主要解决 JS 运行、线程通信和高性能原生能力调用问题。

---

## 七、公共工具模块

### 模块定位

公共工具模块主要为 ASCF 核心模块提供通用基础能力。

### 主要功能

它的主要功能包括：

1. 公共工具函数。
2. 常量定义。
3. 类型定义。
4. 通用协议类型。
5. 通用配置。
6. 日志、错误处理等基础能力。

这一层通常不直接承担业务流程，但会被运行时核心、API 模块、桥接模块、组件模块共同依赖。

---

## 八、应用工作域

### 模块定位

应用工作域主要对应鸿蒙项目中的全局应用配置区域。

### 主要功能

它通常用于管理：

1. 应用级配置。
2. 全局资源。
3. 权限声明。
4. 应用元信息。
5. HSP 相关配置。

这一部分虽然不是 ASCF 运行时逻辑本身，但会影响模块能否正常构建、启动和被其他应用集成。

---

## 九、核心模块整体链路

根据目前的结构，可以将 ASCF 核心模块的整体运行链路理解为：

```text
鸿蒙应用启动
  ↓
主模块接管生命周期
  ↓
创建上下文
  ↓
启动运行时核心
  ↓
加载清单配置
  ↓
加载主包 / 分包
  ↓
预加载 app.js
  ↓
初始化页面路由
  ↓
创建或管理 WebView
  ↓
加载视图层页面
  ↓
建立 JS 与 ArkTS 通信桥接
  ↓
注册并分发 API 调用
  ↓
必要时进入原生桥接模块
  ↓
调用鸿蒙系统能力
  ↓
结果返回给 JS 和页面
```

## 十、问题排查方向

根据当前模块分层，后续排查问题时可以按照现象定位到不同模块。

| 问题现象                             | 优先排查方向              |
| -------------------------------- | ------------------- |
| 应用启动失败                           | 主模块、上下文管理、生命周期管理    |
| 页面白屏                             | Web 容器、页面路由、配置加载    |
| app.js 未执行                       | 预加载、包管理、运行时启动流程     |
| 页面跳转失败                           | 页面路由、标签栏、WebView 管理 |
| JS API 找不到                       | API 模块、桥接层、注册分发流程   |
| JS 调用没有返回                        | 桥接层、原生桥接模块、回调机制     |
| 设备 / 网络 / 媒体 / 存储能力异常            | 对应 API 能力模块         |
| video / map / camera / canvas 异常 | 组件库、同层渲染、原生组件能力     |
| 逻辑层和视图层不同步                       | 原生桥接模块、双线程通信        |
| 构建或导入失败                          | HSP 配置、应用工作域、包管理配置  |

---

## 十一、当前理解总结

目前可以先把 ASCF 核心模块理解为：

```text
主模块 HSP
 负责鸿蒙侧入口、生命周期、Web 容器和对外导出

运行时核心
 负责页面路由、配置加载、app.js 加载、包管理、WebView 管理和桥接通信

组件库
 负责同层渲染组件和复杂原生组件增强

API 能力模块
 负责设备、网络、媒体、存储、UI 等 API 的具体实现

原生桥接模块
 负责 V8 运行环境、双线程通信、JS 与 ArkTS / C++ 桥接

公共工具模块
 负责工具函数、常量、类型和通用基础能力

应用工作域
 负责应用级配置、资源、权限和 HSP 相关配置
```

这说明 ASCF 核心模块并不是单一功能模块，而是一个完整的鸿蒙侧运行时系统。
